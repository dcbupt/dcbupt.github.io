Maven笔记

初体验

安装和配置

下载：windows环境
maven.apache.org/download.cgi 下载Binary zip，别选Source zip

解压缩
添加M2_HOME环境变量（解压缩路径）
PATH添加%M2_HOME%\bin

测试安装成功：
mvn help:system //help为插件名，system为goal
看到最后输出System Properties和Environment Variables表示成功

maven配置文件是conf/settings.xml，建议复制到~/.m2/目录下，这样maven配置只对当前用户生效
在settings.xml里可以自定义本地仓库路径。默认在.m2/repository

maven命令含义：
mvn 插件名:目标名

创建maven项目

命令：mvn archetype:generate ，接一些-D参数可以设置groupId、artifactId等
maven项目目录结构
{maven.proj}
|–pom.xml
|–src
| |–main
| | |–java
| | |–resources
| |–test
| | |–java
|–target
| |–classes

编译运行Maven项目

你要先cd到项目的根目录，一般通过mvn的archetype:generate的项目，根目录下都会生成一个pom文件
这个pom文件是对该maven项目的描述，基于该pom文件可以方便的实现编译运行

这个pom并未指定src目录，生成class的目录等信息，但依旧实现了compile和exec
这是因为该pom继承上级pom，上级pom定义了大量的默认配置
也就是说，绝大多数配置，无需用户手动配置，也就是maven的核心思想：约定优于配置

maven生命周期

当你看到这样一条命令： mvn install ，install其实是maven生命周期的一个phase

什么是生命周期？简单点就是maven执行一系列操作的完整流程。

所以，一个生命周期包含一系列顺序执行的操作，每个操作称为一个phase。

mvn phase1 phase2 …… 是可以的，你可以执行某个phase，那么该Phase所属生命周期之前的phase也会执行。

maven包含三个生命周期：clean\default\site，其中default最常用

clean周期包含的phase：pre-clean -> clean -> post-clean

default周期包含的phase：compile -> test -> package -> install -> deploy
install部署到本地仓库，deploy部署到远程仓库

其实phase这个概念就是抽象出来的，每个phase本质上都对应 mvn 插件:目标形式
phase与插件名目标名的绑定可以在pom.xml的build元素里自由配置

maven坐标

groupId:artifectId:packaging:version

使用maven管理web项目

  1. 使用maven创建web项目：mvn archetype:generate -D一系列参数，熟悉吧，之前就是这么创建项目的，只不过web项目需要特定的参数，具体参数请参考P41

  2. POM文件添加一些项目描述信息：这个没啥说的，进去添加些XML元素来描述项目，并没什么具体的作用

  3. POM文件添加依赖：重点来啦！你要用struts2框架？easy! 走起dependency元素里添加struts2项目的坐标来定位其实struts2框架也就是一个开源项目吧里面有项目打包好的一些类库给你用本质上还是一个项目自然要用到maven的坐标来定位啦

  4. 打包mvn package：这一步执行完毕，你会看到在target目录下有一个压缩文件，文件格式看你POM定义的package内容是啥，一般就war/jar两种选择。打开压缩包，web-inf/lib/目录下你会发现已经集成好的struts2框架~
