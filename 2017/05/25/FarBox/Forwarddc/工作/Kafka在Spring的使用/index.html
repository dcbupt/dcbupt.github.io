Kafka在Spring的使用

发表于 2017-05-25 | 更新于: 2017-11-27 | 分类于 工作 | 阅读次数:
learn by work

Kafka作为一个消息系统，部署在集群中，充当消息中间件的作用。

Kafka分为服务器和客户端两大类，服务器负责消息的转发，客户端负责消息的产生和发送、消息的接受和解析。

Spring支持Kafka，用于Kafka客户端的后台开发，即在后台使用Kafka发送消息和接受消息。

Kafka客户端在Spring中的配置

1、客户端为Producer

KafkaTemplate（Factory\autoFlush\defaultTopic\producerListener） <– Factory <– Prop

逻辑如上，Template发送消息，所以Producer的一切配置都需要关联到它。

Factory注入prop，设置了Kafka服务器地址、groupID等参数配置

defaultTopic指定默认发送消息所属的topic

produceListener由用户实现，监听消息发送成功或失败时的逻辑处理

2、客户端为Consumer

MessageListenerContainer <– {Factory，ContainerProperties} <– {prop}，{Consumer，topic}

Consumer类是自定义的消费者类，负责消费消息，将该类和topic注入一个容器配置类，再把容器配置类注入给消息监听容器，一些配置信息也需要通过Factory类注入到容器

Producer和Consumer的实现

Producer类只需autowired template，调用template的发送接口即可发送消息
注：发送消息的topic在配置template时配了默认值，Kafka服务器和groupID也已经配置了

Consumer类需要实现MessageListener接口的onMessage方法消费消息，消息的topic和Kafka服务器、groupID都已经配置了
