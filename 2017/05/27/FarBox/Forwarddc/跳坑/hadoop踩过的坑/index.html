部署hadoop跳坑之旅

resourcemanager无法通过8088端口访问

3台VM搭建了一个HADOOP集群（一台namenode/jobtracker/master，两台datanode/tasktracker/slave）

配好了后，50070可以访问，但8088不行！

netstat -ano | grep 8088看了一下，发现监听的是ipv6协议，先把server上的ipv6服务关了（改grub）

还是不行！

发现50070是hadoop自己默认配置好的，并不能在配置文件xml中修改（根本找不到这个），但8088作为resourcemanager的web端口是可以配置的

我之前配置的value是hadoop1:8088，hadoop1在我的server的hosts文件中匹配127.0.0.1，所以server是可以访问的，但在宿主机上不行
因为宿主机上识别不了hadoop1,

所以把hadoop1替换为VM的ip就好了

hdfs无法put文件

hadoop fs -put localFile /test
向hdfs文件系统的/test目录下存文件时，显示namenode无法连接datanode

在namenode执行put操作，实际是把文件写到datanode中，所以datanode必须要能找到namenode

在core-site.xml配置中：

所以datanode会到Namenode的hosts找Hadoop1的ip，但namenode的hosts的第一行就是：
Hadoop1 127.0.0.1
所以datanode又找到自己了，就找不到namenode了

所以在namenode中，把hosts的这一行注释掉就好（前提是hosts里已经有了集群中主机名和ip的关系）

集群模式无法启动datanode

在MASTER中./start-all.sh启动集群，但slaves节点的datanode进程无法启动

MASTER和SLAVES互相可以ping通，也已经正确配置

解决办法
删除SLAVES的hdfs/data/路径下的所有内容即可

Win下用idea跑MR（集群在虚机中）

我本以为Win不作为HADOOP节点就不用配HADOOP环境，还是天真了……

在idea中写好MR程序后，执行会出现几个坑：

1、failed to locate the winutils binary in the hadoop binary path
解决办法：下载winutils.exe到你的HADOOP_HOME/bin/

2、java.lang.UnsatisfiedLinkError: org.apache.hadoop.io.nativeio.NativeIO$Windows.access0(Ljava/lang/String;I)Z
解决办法：把我的网盘里的org.apache.hadoop.io.nativeio.NativeIOJAVA文件放到工程java目录下（创建对应的包名）

3、hdfs提示权限异常
解决办法：修改hdfs-site.xml

Win下idea执行的MR任务无法提交到集群

虽然idea执行MR任务成功，但该job并未commit到HADOOP集群，所以无法找到执行过的MR JOB记录

异常说用户无权限
解决办法：
因为windows下的用户名与集群用户名不同导致，修改HDFS文件夹权限：hadoop fs -chmod 777 /{dir}
