活动的生命周期

onCreate-onStart-onResume-onPause-onStop-onDestroy-onRestart
完整生存期：onCreate->onDestroy
可见期：onStart->onStop
交互期：onResume->onPause

app先call onCreate加载布局与监听等，然后call onStart使app可见，再call onResume使得其可以被交互，这三个函数会在app启动时先后调用
当从MainActivity到其他活动时，会先后调用onPause暂停与当前活动的交互、onStop使活动不可见，如果退出程序，会调用onDestroy
如果又back回到MainActivity，会call onRestart

启动模式

活动有四种andorid:launchmode，包括：standard，singletop，singletask，singleinstance
每个APP都会维护一个栈，当启动新的活动时，旧的活动会入栈。

用法：
在活动注册XML中，定义：android:launchMode="singleTask"

默认启动的是standard模式，该模式不区分活动，每次启动的都是新的活动
singletop模式在启动活动时，如果活动位于栈顶，则返回活动，否则创建新的活动
singletask模式对于同一个活动，只会创建一次，当再次访问时，会返回之前创建的活动
singleinstance模式修饰的活动，会用另一个单独的栈来保存。这么做的目的是为了让多个活动能够共享singleinstance修饰的活动。

布局UI

LinearLayout

线性布局
1、定义布局的组件水平/垂直排列
android:orientation="horizontal" 这里也可以是vertical，默认的orientation是horizontal

2、定义组件的对齐方式
android:layout_gravity="left|top" 这里是组件相对布局的对齐方式，注意其与布局的orientation有关

3、设置组件所占宽（高）占比

dp是组件大小的单位，这里虽然为0，但是Android会累加所有layout_weight的值，单个组件所占的宽（高）是其自身的layout_weight的占比

PercentFrameLayout

百分比布局，类似LinearLayout的layout_weight功能

1、导入PercentFrameLayout
该布局是新布局，需要在build.gradle里添加依赖：
compile 'com.android.support:percent:24.2.1'

2、创建布局
i. 因为不是内置布局，所以需要写全路径
<android.support.percent.PercentFrameLayout
ii.定义app命名空间
xmlns:app="http://schemas.android.com/apk/res-auto"
其实任何布局里都会使用的是Android命名空间：xmlns:android="http://schemas.android.com/apk/res/android"

3、组件布局

因为该布局是继承FrameLayout的，所以默认组件都会放在左上角，但可以通过layout_gravity改变位置

自定义布局

android:background="@drawable/back_bg"

2、在其他布局里导入自定义布局
<include layout="@layout/title"/>

tips：如果自定义的布局覆盖了系统自带的标题栏，可以在Activity里将标题栏隐藏

3、为自定义的布局控件添加监听
第二点只能保证在其他的布局文件里导入静态的自定义布局，如果想要为自定义的布局控件添加监听：

因为自定义布局使用LinearLayout，所以继承LinearLayout，构造函数里通过LayoutInflater加载布局文件，并添加控件监听

ii. 在布局文件里导入这个Activity（该Activity对应了自定义布局）

ListView

ListView是最常用控件，因为其可以通过拖动的方式显示超过屏幕的数据

使用ListView：
1、在布局中创建ListView组件

2、在Activity里创建ListView需要显示的数据（通常是一个List）
private List<Fruit> fruitList = new ArrayList<Fruit>();

3、将数据与ListView适配器绑定
FruitAdapter adapter = new FruitAdapter(MainActivity.this, R.layout.fruit_item, fruitList);
FruitAdapter是自己实现的适配器。因为ListView以多行的形式显示数据，适配器作用就是在每一行将数据适配自定义的布局，这里就是R.layout.fruit_item

如果想要加载自定义的ListView子布局，需要在自定义的适配器中重写getView方法，该方法会在新的子布局要显示的时候被调用。
如果View对象为null，就通过LayoutInflater加载子布局，通过返回的View得到子布局的控件，将子布局的控件通过ViewHolder保存起来，这么做是为了在第一次加载子布局的时候就保存控件，避免了重复获取控件影响性能。
如果View不为null，直接取得保存控件的对象。
最后控件调用set方法显示List里的对象的数据即可。

4、将适配器注入到ListView

至此，ListView可以完成加载自定义的子布局和显示数据

RecyclerView

ListView的增强版，和PercentFrameLayout一样也是新出的

1、从support库导入RecyclerView
compile 'com.android.support:recyclerview-v7:24.2.1'

2、创建RecyclerView控件

3、设置RecyclerView的布局方式

这就是RecyclerView有别于ListView的地方，ListView自身控制布局方式，所以只能采用LinearLayout的Vertical方式布局，但RecyclerView将布局方式交给LayoutManager负责，再将其注入给自己，实现自由地设定自己的布局方式
这里设置的是瀑布流网格，按列布局排成三列

4、创建适配器
提供静态内部ViewHolder类专门保存子布局View，通过子布局View不仅可以为View添加监听，还可以为子布局里的控件添加监听，而ListView只能为子布局添加监听。
相比ListView的适配器，具有更完备和强大的接口管理子视图：
