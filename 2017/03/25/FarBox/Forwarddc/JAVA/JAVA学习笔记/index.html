疯狂JAVA讲义学习笔记

面试

JDK JRE

JDK是JAVA开发工具，它不仅提供JAVA程序运行所需的JRE，还提供一系列编译运行工具。JRE是JAVA程序的运行环境，它最核心的就是JVM及核心类库。

CLASSPATH

CLASSPATH保存一些目录和jar文件路径，这些路径是为JAVA程序在编译和运行的时候搜索类而用到的。

动态指定类搜索路径

javac -cp java -cp

JAVA跨平台

JAVA编译后不直接生成机器码，而是生成由JVM负责执行的字节码，JVM是跨平台的。

JVM

JVM是一种用软件模拟出来的计算机（处理器、堆栈、寄存器、指令系统等），用于执行字节码（将字节码转换成底层硬件的机器指令）

堆和栈

堆和栈都是运行时的数据存储区。
栈存放非static的自动变量、函数参数、表达式的临时结果和函数返回值。栈数据的分配和释放由系统完成。
堆存放的数据是由程序中显示分配的（new出来的对象）。

垃圾回收

JAVA堆内存的释放由垃圾回收器执行。System.gc() Runtime.getRuntime().gc()
显示执行垃圾回收，也不一定立即执行，还是由JVM调度。
JAVA对象的finalize方法（继承Object）会在对象被gc前调用。

JAVA WEB

1、目录结构
/WEB-INF/classes/ /WEB-INF/lib/ /WEB-INF/web.xml /xxx.jsp /xxx.html
web.xml：web容器通过该文件配置整个web应用程序

2、Servlet和JSP
Servlet是服务器端处理HTTP请求的基本组成单元。Servlet是符合一定规范的JAVA类，存活在WEB容器中，由容器来控制Servlet的生命周期
JSP是一种使用JAVA的动态语言。往往用于显示处理后的结果。

3、打包部署
JAVA WEB可以通过jar命令打成WAR包，对于TOMCAT，直接存放到//webapps/

EJB

用于开发分布式项目
1、EJB定义了三类BEAN：会话BEAN、实体BEAN、消息驱动BEAN
会话BEAN：每当客户端请求时，EJB容器就会选择一个会话BEAN来为客户端服务
实体BEAN： 数据库中的表记录的映射。从EJB3.0开始，已经被JPA替代
消息驱动BEAN：接收客户端发送的JMS消息后处理，适合异步开发

2、打包部署
同JAVA WEB，只是打包后的目录结构以及部署到JAVAEE服务器的路径有差异

8种基本数据类型

byte、short、int、long、float、double、boolean、char

==和equls的区别

对于基本数据类型，==判断的是值是否相等。
对于引用类型，==判断的是引用是否指向同一个堆地址，equals可以被重写
String是引用类型，其equals方法不能被重写，因为该方法用来判断String值是否相等。

super

如果父类没有定义无参构造方法，但定义了有参构造方法，那么子类的构造方法中一定要显式调用super方法加上正确的参数来调用父类的有参构造方法。
其余情况，子类构造方法可以不显式调用super，super()会被隐式调用。

接口和抽象类

只要包含了抽象方法，就是抽象类。
接口是更抽象的抽象类，其包含的方法全是抽象的，如果包含static属性，则属性值不可被改变
抽象类用来继承，接口用来实现
抽象类和接口都不可以被直接实例化

内部类

静态内部类和外部类的地位一样，在加载外部类的时候就会同时加载静态内部类（因为是静态的），同样静态内部类不能访问外部类的非静态变量
成员内部类在创建时依赖于外部类，创建的方式：Inner inner = t.new Inner();，t是外部类的对象。成员内部类可以访问外部类的所有属性
局部内部类是在方法体中创建的class，不能是static，且不能在方法体外部实例化，类内部只能访问方法体中的final型变量
匿名内部类有两种，一种在方法的参数中直接new的方式创建，另一种即在方法体中通过new的方式创建类。匿名内部类必须是接口的实现类或者是抽象类的子类

BigDecimal

用于实现对浮点数的精准计算

String对象池

为了避免频繁创建和销毁对象，创建了String对象池。如果采用String s = "dc"的方式创建String，JVM会先去对象池中找是否存在值相等的String对象，如果存在返回引用，不存在JVM会new一个String返回，并将其添加到池中。但如果用new来创建String，则跳过从池中取的步骤。

StringBuilder

如果需要大量拼接字符串，使用StringBuilder。因为如果用+会产生大量的字符串对象，造成JVM资源浪费。
StringBuilder和StringBuffer功能类似，前者线程安全，后者不能保证。

StringBuffer.reverse()

拷贝数组的数据

通过System.arraycopy方法实现数组值的拷贝（深拷贝），拷贝数组修改数组不会影响被拷贝的数组

二维数组

JAVA二维数组是由保存一维数组的引用的一维数组组成的，所以二位数组的长度等于一维数组引用的个数，而每个一维数组的长度可以不同。
所以创建二维数组可以不需要指定列长：int[][] a = new int[3][];

Vector和ArrayList

功能类似，但Vector线程安全（大多数方法都被加了同步），ArrayList线程不安全

实现一个具备迭代功能的集合

Swing的事件模型

所有Swing组件都有addXXListener和removeXXListener方法，它们用于注册和取消注册事件监听器。
参数类型都是XXListener接口的实现类，实现的方法的参数都是XXEvent、
监听器注册给组件，当组件的状态改变（事件发生）时，组件通知注册给自己的监听器（回调监听器的方法）

监听器的适配器的作用

注册给组件的监听器往往是一个实现了XXListener接口的匿名内部类，但XXListener接口通常不止一个方法，但程序中只需要实现其中的一个方法，但仍然要实现其他用不到的方法，就产生了代码冗余。
监听器的适配器XXAdapter是一个基本的XXListener接口实现类， 匿名内部类可以new XXAdapter从而省去不必要的方法实现

File

提供最基本的文件基本操作与获取文件数据相关的API，例如：createNewFile、isDirectory、list、getPath、getName

FileInputStream和FileOutputStream

指定文件的输入输出流，输入流通过read方法将字节流存入byte[]，输出流通过write方法将byte[]字节流写入文件

RandomAccessFile

同时具有read和write的能力，可以read字节，也可以write字符，而且通过seek方法可以指定第i个字节处开始读写

Stream

分为字节流和字符流。字节流没什么，就是InputStream和OutputStream
字符流：BufferedReader和BufferedWriter
字符流依赖于字节流，以BufferedReader读取文件为例，其需要注入InputStreamReader（把字节流转化为字符流并指定解码格式），而InputStreamReader需要注入FilerInputStream（提供字节流），其设计模式为装饰者模式

BufferedWriter可以用另一种更简单的实现类代替：PrintWriter

序列化

实现Serializable接口的类的对象可以被序列化和反序列化。
序列化将JAVA对象内存中的二进制数据通过ObjectOutputStream写入文件，反序列化将文件中的数据通过ObjectInputStream读入内存
可序列化的类还需要有一个private final static long serialVersionUID值来标识

Map

1、Map怎么判断key-value对相等与否呢？
判断key相等，需要equals和hashCode方法都返回true

如果将String作为Key，需要注意：尽管new String(“dc”) == “dc”返回false，但equals和hashCode方法都返回true，所以HashMap会把“dc”和new String(“dc”)当作一个对象

判断value相等，只需要equals返回true
2、尽量不要使用可变对象作为Key
结合1，自己想想为什么。改变了Key对象后可能无法从map里找到了！

  • HashMap和Hashtable
    Hashtable比较古老了（从命名看得出），但有其特点：
    线程安全，不允许存入null（key和value都不允许）
    HashMap的特点：
    线程不安全（可以用Collections包装成线程安全的），允许存入null（key和value都允许，但key不能重复）

  • LinkedHashMap
    维护key-value对的插入顺序

  • Properties
    从指定文件中读取key-value或者将Properties里的key-value对存入文件，与IO中的input/outputStream组合使用

  • TreeMap
    根据key的大小（自然排序或Comparator），对key-value排序，方便得到最大最小的key，内部采用红黑树实现
    TreeMap还可以得到大于Key的最小值或小于Key的最大值，返回指定key区间的子key-value等扩展功能
    注：应该保证当两个key相等时，equals方法也返回true

  • WeakHashMap
    HashMap保存key的强引用，所谓强引用，就是不会轻易被gc回收，除非HashMap被gc回收了
    WeakHashMap保存对象的弱引用，除非key被另一个对象强引用了，否则会被gc回收
    例如key只是一个new出来的对象，并没有被具体对象的引用指代，则会被gc（执行System.gc()）

  • IdentityHashMap
    打破了HashMap的规则：判断key相等根据equals和hashCode
    IdentityHashMap判断key相等，需要”key1==key2”，即严格相等！其实判断相等的条件变弱了！
    当new两个一样构造参数的同一类对象时，HashMap会认为两个key相等导致第二个key-value会取代第一次的key-value，但IdentityHashMap会放行！

  • EnumMap
    要求Key均为枚举值，且不能为null！创建时指定Enum class
    保存的key-value对依照key在Enum被定义的顺序，通过keyset遍历Key可以发现

Set

  • HashSet
    equals和hashCode都返回true才认为相等
    如果hashCode相等但equals不等，则会以链表的形式添加到同一个内存地址
    如果equals相等但hashCode不等，也会存入HashSet但打破了SET的规则
    加入到HashSet的元素尽量不要修改

  • LinkedHashSet
    可以按照添加的顺序访问

  • TreeSet
    排序的Set，通过equals判断元素相等，所以equals和比较方法应该逻辑一致
    如果对TreeSet的元素做修改导致大小关系发生变化，但TreeSet不会调整顺序

多线程

进程和线程

进程是处于运行过程中的程序，线程在程序中是独立并发的执行流。
线程与父进程的其他线程共享进程的全局资源。
进程之间不能共享内存，线程间可以共享内存。创建进程要分配内存空间，创建线程不需要，代价小很多。

线程创建和启动

a. 定义Thread类的子类，override run方法。创建该子类对象的start方法启动线程。这种方式创建线程，多个线程无法共享实例变量，因为每一个线程都只执行各自的run方法， 线程间无法通信。

b. 创建实现runnable接口的类，实现run方法，new该类的对象作为参数传给Thread创建线程对象，调用线程对象的start方法启动线程。实现runnable的类作为Thread的target，多个线程可以使用同一个target，所以这些线程可以共享该target的实例变量。

c. 创建实现callable接口的类，实现call方法。通过FutureTask类包装该类， 将FutureTask类的对象作为target来创建Thread并start启动线程，通过FutureTask对象的get方法获得线程执行的返回结果。callable接口有泛型限制，限制返回值类型。
一般采用runnable\callable的方式创建线程。

线程的生命周期

线程得到系统的调度后执行线程，进入运行态。PC基于抢占式调度策略，系统给每个就绪态线程一个小时间段执行线程，之后剥夺线程所占用资源，选择下一个执行线程时考虑线程优先级。（手机采用的是协作式调度策略，只有一个线程调用sleep后才会放弃所占用资源，即由线程主动放弃）。

线程调用sleep方法，或者线程等待获得同步监视器（等待获得对象锁），或者调用wait释放同步监视器（释放对象锁）后等待notify重新获得同步监视器时，线程处于阻塞态。

join线程

在线程A的执行体中调用线程B的join方法后，线程A阻塞，直到线程B执行完毕。

后台线程

又称“守护线程”。如果所有的前台线程死亡，后台线程自动死亡。Thread提供了setDaemon设置为后台线程。前台线程创建的线程是前台线程，后台线程创建的线程是后台线程。主线程是前台线程。

线程让步yield

调用线程的yield方法后，线程从运行态回到就绪态，并让操作系统重新进行一次调度。但通常不建议用yield，一般用sleep

线程优先级

每个线程优先级与创建他的线程相同。main线程具有普通优先级。JAVA定义了三种Thread优先级：MAX\MIN\NORM_PRIORITY，建议使用这三个常量来设置Thread优先级，使得程序便于移植。

线程同步

a. 同步代码块：使用sychronized（obj）同步代码块，obj即同步监视器，任何时刻只有唯一一个获得了同步监视器的线程可以执行同步代码块。推荐使用并发访问的对象作为同步监视器。

b. 同步方法：使用synchronized修饰的方法。同步方法的同步监视器是this。synchronized只能修饰方法（除构造方法）和代码块。

c. Thread执行Sleep\yield不会释放同步监视器。

d. 使用ReentrantLock显示地对线程执行体加锁、释放锁。使用lock时，将lock对象设为并发访问对象的一个成员变量，做到一个并发对象对应一个lock，依然能保证对并发对象的访问是线程安全的。Thread可以对已被ReentrantLock加锁的代码再次加锁，即一段被锁保护的代码可以调用另一个被相同锁保护的方法。

死锁

线程A等待获得同步监视器对象a，线程B等待获得同步监视器对象b，但线程A占据着对象b，线程B占据着对象a，所以线程A和B都在等待对方先释放自己占据的同步监视器，就出现了死锁。

线程间通信

多个线程操作同一个对象时，如何通信？言下之意，当一个线程得到该对象的锁时，如何通过代码控制线程释放锁，等待锁这样的动作，达到和其他线程通信的效果。

1、Wait\notify\notifyAll
属于Object类的方法，由同步监视器synchronized对象调用
wait：导致当前线程等待，直到其他线程调用该监视器的notify方法或notifyAll方法来唤醒该线程，调用wait方法的当前线程会释放对同步监视器的锁定。
notify：唤醒在此同步监视器上等待的单个线程，选择是任意的。只有当前线程放弃对同步监视器的锁定（调用wait方法），才可以执行被唤醒的线程。
notifyAll：唤醒在同步监视器上等待的所有线程。

2、Lock\condition
当代码通过lock对JAVA对象加锁代替synchronized时，不存在同步监视器对象，通过condition对象代替同步监视器对象，condition对象提供了三个方法await\signal\signalAll对应wait\notify\notifyAll
通过lock.newCondition方法创建condition对象

3、BlockingQueue
当生产者线程试图向BlockingQueue放入元素时，如果队列已满，则该线程阻塞。如果消费者线程试图从BlockingQueue里取出元素时，如果队列已空，则该线程阻塞。
BlockingQueue是一个接口。

线程组

基本概念
默认情况下，子线程和创建它的父线程处于同一个线程组内。
一旦某个线程加入了线程组，它将一直处于该线程组内，直到线程死亡。
在创建线程时就可以指定线程组。
线程组可以设为后台（守护）线程组，后台线程组的最后一个线程死亡后，后台线程组自动销毁。

线程组的应用-线程异常处理
线程类提供了两个可以设定异常处理器的方法，由异常处理器处理线程异常。
异常处理器：实现了Thread.UncaughtExceptionHandler接口的类。该接口声明了一个处理异常的方法。
如果线程类没有设定异常处理器，那么由线程组的异常处理方法处理异常（线程组实现了UncaughtExceptionHandler），本身就具备异常处理器的作用。

线程池

基本概念
线程池创建大量空闲线程，接收runnable和callable对象启动线程，线程结束后返回线程池。通过线程池设定最大线程数可以有效控制并发线程的数量，防止系统崩溃。

创建线程池
由Exauctors工厂类的静态方法创建线程池ExecutorService

接受Runnable、Callable对象
通过submit\schedule方法接收对象，执行线程，该方法返回Future对象，可以通过Future get到返回值

关闭线程池
shutdown方法关闭线程池，关闭线程池后，不再接收新的runnable\callable对象，但会执行完已经在线程池里执行的线程。shutdownnow方法会关闭所有正在执行的线程

JAVA8新增的ForkJoinPool线程池实现线程切分并行执行
ForkJoinPool利用多核CPU的物理性能，可以将一个大任务（线程）拆分成多个小任务（线程）并行执行。FJP不接收Runnalble或者Callable，而是接收实现了ForkJoinTask接口（一个实现类带返回值，一个不带返回值）的对象，该对象的compute方法作为线程执行体，调用该对象的fork方法实现多个线程并行执行（通常在compute里new ForkJoinTask子线程，调用子线程的fork方法实现子线程并行执行）。submit方法返回Future对象get到线程返回值

通用线程池
不受shutdown影响的线程池。ForkJoinPool的commonPool静态方法返回一个ForkJoinPool对象，该对象是通用线程池，可以保证submit的线程执行体一定可以执行完毕。

ThreadLocal

如果多个线程对同一个JAVA对象操作，那么这些线程将共享对象的成员变量，但如果在这个JAVA对象里创建ThreadLocal对象，则每个线程将独占这个ThreadLocal对象，即ThreadLocal对象给每个线程创建一个副本，线程对ThreadLocal的操作是对副本的操作，彼此间互不影响。

ThreadLocal支持泛型，通过get\set方法保存数据。

IO

IO分为输入流输出两种，每种有包括字节流和字符流

基本概念

IO流使用装饰者模式，将IO流分层底层节点流和上层处理流。节点流关联物理存储单元，不同节点流虽然存在差异，但可以被包装成统一的处理流。
IO流创建时通常放在try块中，这样在不用时JVM会负责关闭IO流

字节流和字符流

InputStream和Reader是输入流的抽象基类。InputStream和Reader都提供read方法读取数据，区别是：
InputStream读取的是字节流，存入byte[]。Reader读取的是字符流，存入char[]
OutputStream和Writer可以类比上面，不过要注意：
1、OutputStream的write方法既可以输出byte[]中的字节，也可以输出char[]中的字符
2、Writer的write方法直接输出字符串

处理流

作用：隐藏底层设备上节点流的差异，提供更加方便的输入输出方法
处理流的构造参数是已经存在的流，节点流的构造参数是物理存储单元

典型应用：
用PrintStream包装OutputStream写数据，用BufferedReader包装Reader读数据

转换流

转换流负责将字节流转为字符流，包括：InputStreamReader和OutputStreamWriter
InputStreamReader接收InputStream，并转换为字符流，可以被BufferedReader包装来读取数据

推回输入流

PushbackInputStream和PushbackReader具有unread方法，实现将读到的数据推回到缓冲区，实现重复读取数据的功能。

读写其他进程的数据

通过Runtime类获取单例Runtime后调用exec方法输入cmd参数，可以创建一个子进程Process

读取子进程的数据
子进程Process对象提供对外的输出流api，可以通过BufferedReader包装这个流实现进程数据的读取，当然因为BufferedReader处理字符流，还需要转换流包装线程的输出流将其转为字符流

写入数据给子进程
通过PrintSream包装子进程p的输出流

RandomAccessFile

RandomAccessFile构造时直接传入文件的路径，并指定操作文件的方式（读写），其seek方法可以指定读取数据的位置，即从第几个字节处开始读。
注意，这里read的数据以byte[]存储。如果是write，可以直接用字符串。

对象序列化

序列化是把JAVA对象转化为字节序列存储在磁盘中，便于在网络中传输。
类必须实现Serializable接口或Externalizable接口，前者是标记接口，后者需要实现序列化反序列化的规则：writeExternal和readExternal方法

序列化Demo：
假设Class A implements Serializable接口，那么借助ObjectOutputStream和ObjectInputStream的writeObject、readObject方法就可以读写JAVA对象。
注意：ObjectInput/OutputStream都是处理流，所以需要注入节点流

注意事项：
1、反序列化读取的是对象的数据，所以依赖该对象所属类的class文件
2、反序列化机制不需要调用构造函数
3、反序列化恢复对象时，必须按照写入的顺序读取
4、可序列化类如果有父类，父类必须满足：
i、可序列化
ii、不可序列化但提供无参构造函数，但这时父类的成员变量值无法序列化
5、如果可序列化类包含其他类对象的引用，则该引用所属的类必须也可以序列化（递归序列化）

序列化算法：
同一个对象只会在第一次被序列化并生成序列化编号，之后如果还需要序列化（例如该对象被其他对象引用），则直接输出编号。
这保证了：反序列化能得到同一个对象
注意：这样做造成如果对象被序列化之后做了修改，那么这些修改不能体现在反序列化的对象里

自定义序列化：
1、transient：被transient修饰的属性不能被序列化
2、可序列化的类通过实现readObject和writeObject方法，自定义序列化和反序列化的规则。
注意：readObject和writeObject的规则应该相互对应，保证可以正确反序列化
3、实现writeReplace方法，该方法返回的对象会代替原对象被序列化。每次序列化时，JVM都会先调用wirteReplace再调用writeObject。
4、readResolve方法在readObject方法后被执行，该方法返回的对象会作为最终反序列化输出的对象。
注意：所有单例、枚举类，因为构造函数私有，所以在反序列化时往往不能得到同一个对象，就必须实现readResolve保证反序列化对象的正确性。

版本

最好为可序列化的类创建一个private static final long serialVersionUID的属性，值标识class版本。
因为在序列化反序列化双方，需要保证class文件兼容，但随着项目升级，class文件会发生修改造成版本不一致。

NIO

面向流的输入输出系统一次只能处理一个字节，效率不高。NIO采取内存映射文件的方式处理输入输出。

Channel和Buffer是NIO的核心。
Channel对应传统IO的流，不过Channel是面向块，将一块数据映射到内存中。
Buffer是一个容器，本质是一个数组，所有发送给Channel的数据或者从Channel读取的数据都必须先放到Buffer中。

Buffer：
Buffer中有三个概念：Capacity、Limit、Position
Capacity即Buffer创建时的长度，Position对应写入或读取Buffer时的下标。
Limit用于保证读写数据的正确性。创建Buffer或调用clear方法后，Limit等于Capicity。

当写完数据后，Buffer提供flip方法为读取数据做准备。flip方法会将Position设为0，Limit保存Buffer数组里最后一个put元素的下标。Limit之后的元素是不可以被读写的。

常用Buffer：

Channel：
NIO里的Channel按照功能划分，例如FileInput/OutputStream都认为是FileChannel。
所有Channel都必须通过节点流的getChannel方法创建。
Channel最常用的方法：map、read、write
map方法把节点对应的物理存储单元中的数据映射为ByteBuffer，read和write方法读写Buffer。

Charset
从Channel中map的Buffer是ByteBuffer，Charset指定字符集，提供Decoder对象负责解码ByteBuffer为CharBuffer。

文件锁
文件锁用于阻止多线程并发修改文件，通过FileChannel.lock/tryLock获取FileLock，通过FileLock.release方法释放锁
tryLock无法获取文件锁不会阻塞线程。
lock和trylock方法可以指定获取的文件锁是否是共享的。共享锁允许多线程读取文件。

Paths和Files
Paths和Files都是工具类。Paths的工具方法可以返回Path接口对象，代表一个与平台无关的路径。作用类似于之前的File类。
Files包含了大量的工具方法，以Path作为参数，完成文件复制、读取写入内容等功能，而这些操作原本需要通过IO操作才能完成。

FileVisitor
FileVisitor是一个用于遍历文件的接口，SimpleFileVisitor是其基本的实现类，重写该类的方法实现在访问到文件时做需要的逻辑

WatchService
监视指定路径下所有文件发生的事件。Path注册WatchService，WatchService监听事件，并通过take方法取出WatchKey，通过WatchKey找到事件信息WatchEvent。

JDBC

跨平台跨数据库（如果使用标准SQL语句）
JDBC与底层数据库解耦，更换底层数据库不需要修改JDBC代码，只要更换数据库驱动程序

JDBC编程步骤

1、加载数据库驱动程序：Class.forName(com.mysql.jdbc.Driver);
2、获取数据库连接：DriverManager.getConnection(url, user, passwd);
3、创建Statement对象：Connection.createStatement()\prepareStatement(sql)\prepareCall(sql)
4、执行SQL语句：Statement.execute\executeUpdate\executeQuery
executeUpdate执行DML和DDL的SQL，执行DML返回受影响行数，执行DDL返回0
executeQuery执行查询SQL，返回ResultSet
5、通过ResultSet获取结果集
ResultSet的getString(index)可以获取除了blob之外的任何数据
6、关闭ResultSet、Statement、Connection等资源（因为这些类都实现了AutoClosable接口，所以可以把他们的创建都放在try()语句里实现自动关闭）

PreparedStatement

PreparedStatement是Statement的子接口，可以预编译SQL语句，预编译后的SQL语句存储在PreparedStatement对象中，然后可以使用该对象多次高效执行该SQL。
在创建preparedStatement时传入的SQL字符串包含占位符 "?" 参数。
之后通过setXXX(index, value)给占位符赋值。最后调用executeUpdate/Query方法执行SQL

好处：
1、预编译SQL，性能更好。2、防止SQL注入

ResultSet

默认创建的ResultSet的记录指针不可自由移动，且不可通过ResultSet更新结果集
可以在创建Statement或PreparedStatement时，传入ResultSet.TYPE_SCROLL_SENSITIVE和ResultSet.CONCUR_UPDATABLE参数，这样在执行execute后，得到的ResultSet就是可自由滚动、可更新的。
rs.absolute(index)定位到指定记录行，rs.getXXX(index)获取字段值， rs.updateXXX(index, value)更新结果集，rs.updateRow提交更新

处理Blob数据

Blob为二进制类型，用于保存多媒体类的数据（图片音频）

1、通过PreparedStatement向数据库中存入Blob类型的文件（图片或音频）
首先需要将文件转化为流FileInputStream，然后调用PreparedStatement的setBinaryStream函数存入FIS，最后调用PreparedStatement的executeUpdate方法执行SQL插入语句

2、通过ResultSet取出Blob数据
以将Blob数据转化成ImageIcon图片为例：首先通过ResultSet的getBlob(index)方法得到Blob数据对应的Blob对象。然后调用Blob对象的getBytes方法得到对应的byte[]数组，作为参数传给ImageIcon即可得到对应的图片

ResultSetMetaData

通过ResultSet的getMetaData方法可以获取ResultSetMetaData对象，该对象可以访问结果集中包含的字段信息
getColumnCount返回字段个数，getColumnName(index)返回字段名，getColumnType(index)返回字段类型

RowSet

RowSet接口提供了一种离线（不需要保持Connection连接）处理结果集ResultSet的方式。原理是将结果集中的数据读到内存中封装成RowSet对象。
RowSet接口是ResultSet的子接口，RowSet下有多个子接口，包括JdbcRowSet，CachedRowSet，WebRowSet等。

RowSetProvider创建RowSetFactory，RowSetFactory工厂类负责创建RowSet子接口的不同实现

以CachedRowSet为例，通过调用populate方法传入ResultSet导入结果集，之后就可以关闭Connection离线处理。
注：如果CachedRowSet调用updateXXX方法更新了结果集，需要重新获得Connection后，调用CachedRowSet的acceptChanges(Conn)方法更新数据库

网络编程

基础知识

端口：
端口是一个16位的整数，用于表示数据交由哪个通信程序处理。它是一种抽象的软件结构，包含一些数据结构和IO缓冲区。
端口号可以从0到65535，01023为公认端口，它们紧密绑定一些特定的服务。102449151为注册端口，应用程序通常应该使用这个范围内的端口。49152~65535为动态/私有端口，这些端口是应用程序使用的动态端口，应用程序一般不会主动使用这些端口。

基础类

使用InetAddress
InetAddress类代表IP地址，该类提供了两个静态方法获取InetAddress实例。提供了实例对应的IP地址和主机名的方法。

使用URLDecoder和URLEncoder
URLDecoder和URLEncoder用于完成普通字符串和MIME字符串之间的相互转换。MIME字符串是系统将URL地址里包含的非西欧字符串转换成的特殊字符串。URLDecoder将MIME字符串转换成普通字符串，URLEncoder将普通字符串转换成MIME字符串。

URL、URLConnection和URLPermission
URL由协议名、主机、端口和资源组成。URL有多种构造方法，URL对象的openStream方法返回InputStream用于读取URL资源。

创建URL连接并发送请求、读取URL引用的资源步骤：
1、通过URL对象的openConnection创建URLConnection对象。
2、设置URLConnection的参数和普通请求属性。
3、如果只发送get请求，使用connect方法，如果发生POST请求，需要获取URLConncetion实例对应的输出流来发送请求参数。
4、访问响应的头字段或通过输入流读取资源的数据。
注：如果程序需要发送POST请求，需要先设置doIn和doOut两个请求头字段的值为true。

基于TCP协议的网络编程

TCP/IP基础：

TCP/IP协议是一种可靠的网络协议，它在通信的两端各建立一个Socket，从而在通信的两端之间形成网络虚拟链路。JAVA使用Socket对象代表两端的通信端口，并通过Socket产生IO流来进行网络通信。

IP协议使Internet成为一个允许连接不同类型的计算机和不同操作系统的网络。

TCP协议被称作一种端对端协议。发送方将数据包按序排列，接收方按序还原。重发机制保证了可靠性。

使用ServerSocket创建TCP服务器端：

ServerSocket对象用于监听来自客户端的Socket连接。JAVA程序通常会通过循环不断调用ServerSocket的accept方法接收Socket请求。
创建ServerSocket没指定IP地址使会绑定到本机默认的IP地址。

使用Socket进行通信

客户端使用Socket对象连接服务器，创建Socket对象需要指定服务器IP地址和 端口号。Socket提供了两个方法来获得输入流和输出流进行读写IO操作。Socket提供了当网络连接（创建Socket）、读取操作超过设定时间后抛出异常的方法。

加入多线程

在使用传统BufferedReader的readLine方法读取数据时，在该方法成功返回之前，线程被阻塞，程序无法继续执行。因此，服务器端应该为每个Socket单独启动一个线程负责与该Socket代表的客户端进行通信。

客户端读取服务器端数据的线程同样会被阻塞。所以应该在客户端也单独启动一个线程负责读取服务器端数据。

半关闭的Socket

网络通信中不能关闭输出流来表示输出结束，因为关闭输出流意味着该输出流对应的Socket也将随之关闭，导致程序无法再从该Socket中读取数据。
Socket提供了两个半关闭的方法，只关闭Socket的输入流或输出流。

使用NIO实现非阻塞Socket通信

使用NIO API可以让服务器端使用一个或有限几个线程来同时处理连接到服务器端的所有客户端。

原理：
通过Selector对象汇聚多个客户端的SocketChannel和一个服务器端ServerSocketChannel。通过Selector统一管理这些Channel，通过selector对象的select方法判断是否有IO请求（IO请求包括ServerSocketChannel.accept，SocketChannel.read，SocketChannel.write）的Channel（无IO请求会阻塞线程）。

服务器端使用NIO处理多客户端Socket：
1、创建ServerSocketChannel对象并设置为非阻塞模式。
2、将该对象注册给selector。
3、selector对象调用select方法判断是否注册给它的Channel有IO请求。
4、得到有IO请求的Channel将其注册到Selector以及其他逻辑处理。

注：SocketChannel、ServerSocketChannel注册给Selector时，需要指定mode。mode有：accept(只提供给ServerSocketChannel)、read、write。Selector根据注册的mode来判断Channel是否有IO请求。

Selector、SocketChannel实现单线程监听多客户端

Selector是一个监听器，监听SocketChannel和ServerSocketChannel。ServerSocketChannel对应ServerSocket，SocketChannel对应Socket。Selector用SelectionKey保存每一个监听的Channel。ServerSocketChannel对应的SelectionKey只具备accept功能，即只会接收SocketChannel，SocketChannel对应的SelectionKey具备read&&write功能。

服务器端：创建ServerSocketChannel，注册到Selector，判断Selector.select是否有需要IO的Channel，遍历SelectionKeys，判断是accept类型还是read类型，accept类型就将accept的SocketChannel注册到selector，read类型就读取数据。

综上，Selector保存了一个服务器端ServerSocketChannel和多个客户端SocketChannel，服务器端Channel负责accept客户端的Channel后注册到Selector，客户端Channel负责取数据。只需要selector不断select，即监听多个Channel是否有IO，就实现了单线程对多个客户端请求的响应。

使用JAVA 7的AIO实现非阻塞通信

AIO概念：
IO操作分为两步：1、程序发出请求。2、完成实际的IO操作。阻塞、非阻塞IO是针对第一步划分的，如果发出IO请求会阻塞线程，就是阻塞IO。同步和异步IO的区别在第二步。如果实际的IO操作由操作系统完成，再将结果返回给应用程序，就是异步IO。阻塞IO和非阻塞IO都是同步IO。异步IO本身是由操作系统完成IO再返回结果，其本身就是非阻塞IO。

使用AsynchronousServerSocketChannel
1、调用open静态方法创建AsynchronousServerSocketChannel对象。
2、调用bind方法绑定IP和监听端口号。
3、调用accept方法接受连接请求。accept方法会传入实现了CompletionHandler接口的对象，操作系统通过调用该对象的complete方法完成业务逻辑。客户端AsynchronousSocketChannel对象的read、write方法同样是IO操作，也可以传入CompletionHandler对象。

基于UDP协议的网络编程

预备知识

UDP协议是一种不可靠的网络协议。
它在通信两端各建立一个DatagramSocket，使用DatagramPacket代表发送、接收的数据报。

使用DatagramSocket发送、接收数据

DatagramSocket不产生IO流，只负责发送和接收数据包。服务器端的DatagramSocket必须指定端口。

DatagramSocket并不知道将该数据报发送到哪里，而是由DatagramPacket自身决定数据报的目的地。

DatagramSocket和DatagramPacket的关系类似于码头和集装箱的关系，DatagramSocket作为码头，不知道数据报发到哪里也不知道接收的数据报来自哪里，只负责发送和接受。DatagramPacket作为集装箱，知道数据报发送到的目的地和接收到的数据报的源地址。

UDP协议是面向非连接的，所以接收者并不知道每个数据报由谁发送过来，但程序可以调用DatagramPacket的方法获得发送者的IP地址和端口。也可直接获得SocketAddress对象，该对象封装了IP和端口。

使用MulticastSocket实现多点广播

DatagramSocket只允许数据报发送给指定的目标地址，而MulticastSocket可以将数据报以广播方式发送到多个客户端。

IP多点广播的思想是设置一组特殊网络地址作为多点广播地址，每一个多点广播地址都被看做是一个组，当客户端需要发送、接收广播信息时，加入该组即可。这些特殊网络地址范围是224.0.0.0~239.255.255.255

创建MulticastSocket时，需要指定多点广播地址。当Multicast把一个DatagramSocket发送到多点广播IP时，该数据报被自动广播到加入该地址的所有MulticastSocket。创建接收用的MulticastSocket必须要指定端口。

使用代理服务器

代理服务器提供如下两个功能：
突破自身IP限制，对外隐藏自身IP地址。突破IP限制包括访问国外受限站点，访问国内特定单位、团体的内部资源。
提高访问速度，代理服务器提供的缓冲功能可以避免每个用户都直接访问远程主机，从而提高客户端访问速度。

直接使用Proxy创建连接：
Proxy类代表一个代理服务器，可以在打开URLConnection连接时指定Proxy，创建Socket连接时也可以指定Proxy。创建Proxy对象需要指定代理使用的协议（HTTP\SOCKS）和代理IP地址。

使用ProxySelector自动选择代理服务器：
直接使用Proxy对象，每次打开URLConnection或者建立Socket都需要传入该对象，十分麻烦。如果希望每次打开连接时总是具有默认的代理服务器，则可以借助于ProxySelector来实现。

ProxySelector类代表一个代理选择器，可以对多种协议如HTTP\FTP等进行分别设置，还可以设置不需要通过代理服务器的主机地址。

ProxySelector本身是一个抽象类，可以继承该类来实现自己的代理选择器。该抽象类设计了两个方法,select方法在产生相应协议的连接时被调用来查找代理地址。connectFailed完成代理失败后的处理。

JAVA为ProxySelector提供了一个默认实现类DefaultProxySelector。该类的Select方法会根据系统属性来决定使用哪个代理服务器。该方法检查系统属性与URL之间的匹配，然后决定使用相应的属性值作为代理服务器。
