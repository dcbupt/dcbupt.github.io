工厂模式

工厂模式用于创建具体对象（产品）。而指向具体对象的，往往是一个抽象的引用。
工厂模式的价值体现在：具体对象与抽象对象的解耦。依赖工厂模式，我无需关心创建了哪个具体的对象，我只需要通过抽象对象的引用来使用它！
工厂模式分为简单工厂、工厂方法和抽象工厂。

装饰者模式

装饰者模式用于扩展方法的功能。被扩展功能的对象称为被装饰者，提供功能扩展的对象称为装饰者。
所谓扩展，意味着扩展的功能依赖于之前的功能！这一点很重要！

  • 提出问题
    类A继承了Dad类并实现了方法a，但a的功能有限，我想对方法a进行扩展。

  • 装饰者模式的做法
    可以定义一个Son类继承Dad，而所有装饰者类都是Son的子类。Son类的方法a还是抽象的，但子类都实现（扩展）了方法a。
    子类怎么扩展方法a呢？通过组合！
    Dad对象的引用作为子类的一个属性，在构造方法中传入需要对其功能进行扩展的子类（类A），然后在方法a的实现中，也许在某处会调用A的a方法，并增加一些额外的功能实现。
    假设装饰者类是B\C\D类，那么当你看到这样的代码时，说明用了装饰者模式：
    D d = new D(new C(new B(new A())));
    D装饰了C，C装饰了B，B装饰了A。

  • 启发

    装饰者模式通常用于对一个基本方法进行扩展，装饰者依赖于被装饰者提供的一个基本实现，在此基础上添加自己的功能

单件模式

有些对象在项目中只需要实例化一次，所以需要对此类对象应用单件模式。

实现一个单例：
创建一个私有的单例对象、一个私有构造函数用于在内部实例化对象
创建一个public static的方法用于返回单例，逻辑是：
1、如果私有单例对象 == null，调用私有构造方法new一个返回
2、如果 != null，直接返回私有对象

如果在多线程中创建单例对象，因为得到单例的方法未同步，可能会造成单例对象被new了多次，三种解决办法：
1、获取单例的方法加同步
2、声明私有单例对象时就new出来，获取方法里直接返回
3、在获取单例的方法中，只在单例为null时，对创建单例的代码加入同步块，这么做减少了方法中的同步代码块的范围。
