剖析‘对象a=对象b’

发表于 2017-03-08 更新于 2022-05-01 分类于 深度系列 热度: ℃ 评论: ℃
字数： 1.2k 阅读时长 ≈ 2 分钟
JAVA中对象不能乱指的

结论置顶

原理：

对于自定义的类的对象，执行a=b时，是让a的栈指向b的堆
对于基本类型和String，执行a=b时，是让a的堆复制b的堆中的值
对于函数的参数传递，执行形参a=实参b

结论：

对于自定义的类的对象，a=b后，b的改动会同步到a（前提b已经new了）
对于基本类型和String，a=b后，b的修改不会影响a
对于自定义的类的对象作为形参，函数体里的改动会影响到实参
对于基本类型和String，函数体里的改动不会影响到实参

剖析

在java中，当使用 a = b （a、b均为JAVA对象）时，一定要慎重！
首先，你要保证，b对象已经new了！不然如果b=null，那么a=null，且不可逆（就算之后new了b）！为什么呢？

a = b的过程是：让a对象的栈指向b对象的堆！栈用来保存存储对象数据的堆地址。
这就是每个人挂在嘴边的那句：JAVA是值传递的终极奥义！
如果指向的是其他对象的栈，那就是指针传递！

下面假设几种情况：

  • b = c
    如果在a的栈指向b的堆之前new过b了，那么b=c不会影响a，只是b的栈会指向c的堆，此时b和c共享c的堆，而a独享b的堆。所以一旦让对象a指向对象b，那么对象a将会一直盯着b的堆，只有b的堆中的数据改变了才会影响a。
  • a = b之前并未new b
    这样会使得a并未真正指向一个堆，即a=null。就算之后new b，也只是b的栈指向了自己的堆（内存分配一块区域保存b对象的数据），但a的栈还是没有指向堆，a仍为null。

引申

当a对象作为参数传入方法中的时候，其实是值传递的！
啥意思？
调用method函数时，其实是这么做的：b=a！
因为b指向了a的堆，所以在函数体里，对b做的修改等效于对a的修改！
看起来就像传进来的是a。但实际上，`是用一个没有堆地址的引用b指向了a的堆

然而当A是基本数据类型或者String时，事情看上去并不是那么简单！

先说结论，如果A是基本数据类型或String类，那么方法里对a的修改不会生效！
原因如下：
当调用函数时，b=a并没有让函数的形参b指向a的堆，b有自己的堆！然后把a的堆里的数据拷贝到自己的堆中了！
所以：函数体里对b的操作只改变了b的堆，a的堆压根没动啊！
