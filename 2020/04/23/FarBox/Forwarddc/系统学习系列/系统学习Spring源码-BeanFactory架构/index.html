系统学习Spring源码-BeanFactory架构

前言

Spring目前使用的bean工厂是DefaultListableBeanFactory，围绕该工厂的类图架构，可以窥探Spring是如何一步步勾勒出这个功能齐全的bean工厂

BeanFactory架构

基础类

BeanFactory

  • bean工厂的顶层接口，就是一个简单工厂，提供获取bean实例的工厂方法getBean

ConfigurableBeanFactory

  • 扩展BeanFactory
  • 可配置的bean工厂，bean工厂的扩展能力在此基础上构建
  • 在BeanFactory的基础上增加了一些配置bean工厂的方法，例如为bean工厂添加属性编辑注册器PropertyEditorRegistrar、设置BeanClassLoader、添加bpp
    • 可配置的bean工厂体现在对bean加载过程的定制化

AutowireCapableBeanFactory

  • 扩展BeanFactory
  • 该工厂专注于创建支持依赖注入的bean实例
    • 定义了创建bean实例接口，需要创建的bean已经完成依赖注入
    • 定义了对实例化的bean实现依赖注入的接口
    • 定义了bean初始化相关接口，也是服务于在bean初始化阶段的依赖注入

DefaultSingletonBeanRegistry

  • 单例bean注册中心，保证单例bean在创建中和创建后的全局唯一性
  • 内部维护一些map缓存，在单例bean创建中和创建后管理单例bean

FactoryBeanRegistrySupport

  • 扩展DefaultSingletonBeanRegistry
  • 支持通过工厂bean获取单例

BeanDefinitionRegistry

  • beanDefinition注册中心
  • 提供BeanDefinition的注册和查询接口，bean工厂实现该接口就会基于beanDefinition对bean定义的统一封装来加载bean

核心类

AbstractBeanFactory

  • 可配置的基础bean工厂
    • 实现了BeanFactory的工厂方法getBean，框定了bean加载的主流程
      • 例如保证了加载单例bean的全局唯一性、支持工厂bean、前置加载depends-on的beans、循环依赖检测等
    • 实现了ConfigurableBeanFactory，工厂可配置
      • 提供bpp、PropertyEditorRegistrar的注册功能
    • 继承FactoryBeanRegistrySupport，支持在bean工厂中注册单例bean
    • 提供模板方法createBean，由子类负责创建bean实例
      - 子类通过`AbstractBeanFactory`的可配置能力（实现`ConfigurableBeanFactory`），拿到bpp、PropertyEditorRegistrar和ClassLoader等变量，定制bean实例的创建过程

AbstractAutowireCapableBeanFactory

  • 可配置、可自动注入的bean工厂，具备完整的bean加载、注册能力
    • 继承AbstractBeanFactory，本身是一个可配置的bean工厂
    • 实现AutowireCapableBeanFactory，创建bean实例时支持依赖注入

DefaultListableBeanFactory

  • 可配置、可依赖注入、可枚举的bean工厂
    • 实现ListableBeanFactory，可枚举bean工厂管理的对象
      • 提供一些byName、byAnnotation对bean工厂管理的bean实例、beanNames、beanDefinition做枚举（查询）的方法
    • 实现BeanDefinitionRegistry，提供对beanDefinition注册、枚举（查询）的能力
    • 继承AbstractAutowireCapableBeanFactory，本身是一个可配置、可自动注入的bean工厂

总结

DefaultListableBeanFactory是Spring默认使用的bean工厂，它是beanDefinition和bean的容器，提供byName、byType、byAnnotation等多种方式枚举bean实例、beanNames、beanDefinition的能力。同时具备bean工厂的基础能力，可以基于注册的beanDefinition加载bean，支持加载过程中的依赖自动注入。并提供一些bean工厂扩展能力，支持自定义bpp、PropertyEditorRegistrar，使用它们定制bean加载过程

DefaultListableBeanFactory不负责解析bean配置生成beanDefinition。XmlBeanFactory作为它的子类，虽然通过注入XmlBeanDefinitionReader和ClassPathResource，具备解析XML的bean定义生成beanDefinition的能力，但这超出了bean工厂的职责范围，所以它最后被废弃了，而是由Spring容器ApplicationContext持有与bean配置对应的BeanDefinitionReader完成解析和创建beanDefinition，然后注册给DefaultListableBeanFactory
